Model Control Panel LLM backend for the preference engine

The preference worker and the opinion playground pull this config live — no restart needed. Use the hosted DeepSeek API, or point base url at a local Ollama instance (http://localhost:11434/v1) and set the model name (e.g. qwen2.5:14b).

Evolution tuning

α = how fast affinities move per observation · seed band = max drift for SOUL-seeded likes (prevents flips) · trait drift = max OCEAN drift from baseline.

Mood granularity

The mood engine scores small "rounds" of ~this many user↔agent exchanges, so mood responds within a chat instead of treating a whole sitting as one point. Smaller round = faster/twitchier, larger = smoother (applies to newly-synced activity). Lookback = how many recent rounds the mood rules evaluate against (this is the "Recent rounds" window).

Emotion scoring

The LLM backend scores each round's user & agent emotions (over the taxonomy below) and the agent's moods in one call — an uncensored local model handles roleplay/adult content best. Changing the backend takes effect on the worker's next poll (no restart needed for switching; editing the worker code does require a restart).

LLM mood votes

Each recent round where the LLM scores a mood ≥ threshold casts weight votes for it, summed across the lookback window and added to the rule votes — so the LLM can carry a mood the rules missed.

Low-signal emotions

Check emotions to filter OUT of the top emotions (some surface on nearly every message). Excluding them lets the distinctive emotions drive moods. A mood rule targeting a checked emotion won't match it. List is driven by the taxonomy above.
